Workflow

Four steps to your executable

01

Place py_builder.pyw in your project folder alongside your scripts

02

Select any .py or .pyw file from the auto-detected list

03

Build with one click — dependencies auto-install if needed

04

Done. Your .exe appears in the same folder

Capabilities

Built for developers who ship

AST Import Scanning

Parses your script at the syntax-tree level to detect every import and installs missing pip packages before the build begins.

Console Auto-Detection

.pyw scripts build with --noconsole for silent GUIs. .py scripts keep the terminal window. No manual flags needed.

Dual Log Views

Build Log shows clean PyInstaller output with a summary. All Log captures startup diagnostics, dependency checks, and every warning.

Auto-Clean

Removes build/, dist/, and .spec files after each successful build. Optional toggle keeps things tidy.

Live Folder Watcher

Drop a new script into the folder and it instantly appears in the list. The watcher scans every second.

Zero-Setup Launch

Auto-installs PyInstaller on first run. Place the script in any project, double-click, and build. No pip install step.

Get Started

Down and running in seconds

Requirements

Windows 10/11 · Python 3.10+

# Clone and enter the directory
git clone https://github.com/jlaiii/Py-Builder-Automated-Python-to-EXE-GUI.git
cd Py-Builder-Automated-Python-to-EXE-GUI

# Run -- PyInstaller auto-installs on first launch
python py_builder.pyw

Tip: Place py_builder.pyw in any project folder and double-click it. It auto-detects all scripts in that directory.
